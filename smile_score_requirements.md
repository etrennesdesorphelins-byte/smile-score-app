# Smile Score ブラウザアプリ 要件定義書

作成日：2026-07-13  
対象：オープンキャンパス展示用・学生卒業研究用プロトタイプ  
実装想定：VS Code + Claude Code 実装、Codex review & 修正

---

## 1. 目的

本アプリは、オープンキャンパス来場者に **MediaPipe Face Mesh / Face Landmarker による顔ランドマーク検出技術を体験してもらう** ことを目的としたブラウザアプリである。

主機能として、Webカメラ映像から顔ランドマークをリアルタイムに表示し、さらに笑顔時の顔ランドマークおよび画像特徴量から **Smile Score 100点満点** を算出する。

本アプリの点数は、医学的・心理学的診断や魅力度評価ではなく、**MediaPipeで取得したランドマーク特徴量に基づく教育・展示用の参考スコア** として扱う。

## 5.2 リアルタイム顔ランドマーク表示・録画機能

### 目的

MediaPipe Face Mesh / Face Landmarker の機能を視覚的に示す。

### 必須機能

- Webカメラ映像をリアルタイム表示する
- 顔ランドマークを映像上に重ねて表示する
- 顔メッシュ、輪郭、目・口周囲のランドマークを表示する
- 録画開始・停止ボタンを設置する
- 録画中であることを画面に表示する
- 録画終了後、以下を出力できるようにする
  - 動画ファイル：`.webm` 推奨
  - ランドマーク座標CSV：`.csv`

### CSV出力項目

| 列名 | 内容 |
|---|---|
| frame | フレーム番号 |
| timestamp_ms | 撮影開始からの経過時間 |
| face_detected | 顔検出の有無 |
| image_width | 映像幅 |
| image_height | 映像高 |
| x_0, y_0, z_0 ... | 各顔ランドマーク座標 |

### 操作

- 「録画開始」ボタンで動画・CSV記録を開始
- 「録画停止」ボタンで記録を停止
- 停止後に動画・CSVをダウンロード可能にする

### 補足

ブラウザアプリのため、Python版のように任意のローカルフォルダへ自動保存するのではなく、原則として **ユーザー操作によるダウンロード** とする。

## 5.3 笑顔採点機能

### 目的

基準顔と笑顔時の顔ランドマーク差分からSmile Scoreを算出し、笑顔の特徴を可視化する。

### 基本フロー

1. カメラ画面を表示する
2. 顔全体を画面上の枠内に入れるよう案内する
3. 顔位置・顔サイズ・顔向きが条件を満たしたら撮影可能とする
4. 「基準顔を撮影」ボタンを押す
5. 3秒カウントダウン後、基準顔を2秒間撮影する
6. 基準顔撮影が成功したことを表示する
7. 「笑顔を採点」ボタンを押す
8. 3秒カウントダウン後、笑顔を2秒間撮影する
9. 代表フレームを表示する
10. 「この画像で採点する」または「撮り直す」を選択する
11. 採点に進む場合、Smile Scoreを算出する
12. 採点結果画面へ遷移する

### 撮り直し仕様

- 来場者が「撮り直す」を選んだ場合、再度3秒カウントダウン後に笑顔2秒間撮影を行う
- 撮り直し回数に制限は設けない
- 基準顔も撮り直せるボタンを設ける

---

## 5.4 顔を枠内に入れる機能

### 目的

採点のばらつきを減らし、顔の位置・大きさ・向きをある程度統制する。

### 表示

- カメラ映像上に顔位置ガイド枠を表示する
- 条件を満たす場合は枠を緑色、満たさない場合は赤色または黄色で表示する

### 判定項目

| 判定項目 | 条件例 |
|---|---|
| 顔検出 | 連続して一定フレーム以上検出できる |
| 顔中心 | 顔中心がガイド枠内にある |
| 顔サイズ | 顔幅・顔高が指定範囲内である |
| 頭部左右傾き | 顔のロール角が大きすぎない |
| 顔向き | 顔が正面から大きく外れていない |
| ランドマーク品質 | 主要ランドマークが検出できている |

### 案内文例

- 「顔を枠の中に入れてください」
- 「もう少しカメラに近づいてください」
- 「もう少しカメラから離れてください」
- 「正面を向いてください」
- 「その位置でOKです」

## 5.5 Smile Score 算出機能

### 総合点

Smile Scoreは100点満点とする。

| 項目 | 点数 |
|---|---:|
| 口角挙上 | 25点 |
| 口幅の拡大 | 20点 |
| 頬・目周囲の変化 | 25点 |
| 左右対称性 | 15点 |
| 笑顔の安定性 | 10点 |
| 上歯の適度な露出 | 5点 |
| 合計 | 100点 |

### 5.5.1 口角挙上：25点

基準顔と笑顔時を比較し、左右口角がどの程度上方へ移動したかを評価する。

実装イメージ：

```text
口角挙上量 = 基準顔の口角y座標 - 笑顔時の口角y座標
正規化値 = 口角挙上量 / 顔高
```

### 5.5.2 口幅の拡大：20点

基準顔と笑顔時を比較し、左右口角間距離がどの程度広がったかを評価する。

```text
口幅拡大率 = 笑顔時の左右口角距離 / 基準顔の左右口角距離
```

### 5.5.3 頬・目周囲の変化：25点

頬の挙上、目の縦幅の軽度狭小化などを用いて、目元を含む自然な笑顔らしさを評価する。

評価例：

- 下眼瞼・頬周囲ランドマークの上方移動
- 眼裂縦幅の変化
- 口角変化と目周囲変化の組み合わせ

### 5.5.4 左右対称性：15点

左右の口角挙上量、口角移動量、目周囲変化の左右差を評価する。

```text
左右差が小さいほど高得点
```

ただし、顔には元々非対称性があるため、基準顔との差分を用いる。

### 5.5.5 笑顔の安定性：10点

笑顔撮影2秒間の連続フレームから、Smile Score候補値の変動、顔検出の途切れ、顔位置の安定性を評価する。

評価例：

- 2秒間の総合スコア変動が小さい
- 顔が枠内に維持されている
- ランドマーク検出が途切れない

### 5.5.6 上歯の適度な露出：5点

口唇内側領域を画像処理で抽出し、上前歯に相当する白色領域が適度に存在するかを補助的に評価する。

点数例：

| 判定 | 点数例 |
|---|---:|
| 歯が見えない | 0〜1点 |
| わずかに見える | 3点 |
| 上前歯が自然に見える | 5点 |
| 見えすぎる・誤検出が強い | 2〜3点 |

### 注意

歯の露出判定は、照明、歯の色、歯列、矯正装置、口紅、口腔内の影、カメラ画質などに影響されやすい。そのため、配点は5点に留め、結果説明でも補助指標として扱う。

## 5.7 アドバイス文章生成機能

### 目的

点数が低かった項目に対して、来場者が傷つきにくい表現で改善ポイントを示す。

### 仕様

- 低得点項目を1〜2個抽出する
- 良い点を先に記載する
- 改善点は前向きな表現にする
- 医学的診断や容姿評価に見える表現は避ける

### 文例

#### 口角挙上が低い場合

```text
口元の表情は検出されています。さらに口角を少し外上方へ引き上げるように笑うと、笑顔の特徴がよりはっきり検出されやすくなります。
```

#### 頬・目周囲が低い場合

```text
口角はよく動いています。さらに頬を軽く持ち上げるように笑うと、目元にも笑顔の特徴が出やすくなります。
```

#### 左右対称性が低い場合

```text
笑顔の動きはしっかり出ています。カメラに対して正面を向き、左右の口角を同じくらい上げる意識をすると、より安定して検出されやすくなります。
```

#### 上歯露出が低い場合

```text
口角はよく上がっています。さらに明るい印象を出したい場合は、上の前歯が少し見える程度に笑うと、口元の表情が伝わりやすくなります。
```

## 5.8 得点ランキング機能

### 目的

オープンキャンパス展示としての参加性・楽しさを高める。

### 表示内容

- ベスト10まで表示
- 表示項目
  - 順位
  - 顔写真
  - Smile Score総合点
  - 登録日時

### 登録確認

採点結果画面で以下を表示する。

```text
ランキングに登録しますか？
顔写真と点数をランキング画面に表示することに同意します。

[登録する] [登録しない]
```

### 必須仕様

- ランキング登録は同意制とする
- 登録しない場合、顔写真と点数はランキングに保存しない
- 個別削除機能を設ける
- 全件削除・リセット機能を設ける
- イベント終了後に削除する旨を画面に表示する

### 表示すべき注意文

```text
ランキング用の顔写真と点数はイベント中のみ使用し、イベント終了後に削除します。
ランキングへの登録は任意です。登録しなくても採点結果は確認できます。
```

### 保存仕様

ブラウザアプリでは、ランキングデータは以下のいずれかで保存する。

| 方式 | 内容 | 推奨度 |
|---|---|---:|
| IndexedDB | ブラウザ内に画像Blobと点数を保存 | 高 |
| localStorage | 画像Base64と点数を保存 | 中。ただし容量制限に注意 |
| サーバー保存 | Node.js等で画像・JSON保存 | 展示PC単体なら不要 |

初期版では **IndexedDB保存** を推奨する。

## 5.9 概要説明ページ

### 目的

アプリの根拠、点数化の考え方、注意事項を来場者・学生・教員に説明する。

### 構成

1. このアプリについて
2. MediaPipe Face Mesh / Face Landmarkerとは
3. Smile Scoreの考え方
4. Smile Scoreの6項目
5. 参考にした研究・資料
6. 注意事項

### 注意事項文

```text
本アプリのSmile Scoreは、MediaPipe Face Mesh / Face Landmarkerで取得した顔ランドマークと画像特徴量に基づく教育・体験用の参考スコアです。
心理状態、性格、魅力度、健康状態、表情筋機能を医学的・心理学的に診断するものではありません。
照明、顔の向き、カメラ距離、眼鏡、マスク、歯の見え方、個人の顔貌によって点数が変動することがあります。
ランキング用の顔写真と点数はイベント中のみ使用し、イベント終了後に削除します。
今回は展示用データとしてのみ扱い、研究解析用データとしては使用しません。
```

## 5.10 管理・リセット機能

### 必須機能

- ランキング個別削除
- ランキング全削除

誤操作防止のため、全削除時は確認ダイアログを表示する。

```text

```

## 6.1 動作環境

| 項目 | 要件 |
|---|---|
| OS | Windows 10/11 推奨 |
| ブラウザ | Google Chrome / Microsoft Edge 最新版推奨 |
| カメラ | Webカメラ必須 |
| ネット接続 | 初回ライブラリ取得時に必要。展示時はローカル配信推奨 |
| 実行方法 | VS Codeからローカル開発サーバーを起動 |

## 6.2 パフォーマンス

- カメラ映像は可能であれば30fps程度
- Face Landmarkerはリアルタイム描画が可能な設定にする
- 採点処理は撮影後3秒以内に結果表示できることを目標とする

## 6.3 ユーザビリティ

- 高校生でも直感的に使えるUI
- 操作ボタンは大きく表示する
- 説明文は短く、画面遷移を明確にする
- 低得点でも否定的表現を避ける

## 6.4 プライバシー

- ランキング登録は同意制
- ランキングに登録しない場合は顔写真を保存しない
- イベント終了後にランキングデータを削除する旨を表示
- 今回は展示用データとしてのみ扱い、研究解析用データとしては使用しない
